# SightFi 技术架构与分层规范（Architecture）

> 更新时间：2026-03-03  
> 目标：保证结构稳定、命名统一、协作成本低。

## 1. 分层结构

| 层级           | 路径                      | 职责                                     | 禁止事项                 |
| -------------- | ------------------------- | ---------------------------------------- | ------------------------ |
| Web App        | `apps/web/src/app`        | 路由装配、布局壳层、全局状态注入         | 直接请求第三方服务       |
| Web Features   | `apps/web/src/features/*` | 按业务域组织页面与组件                   | 跨域直接引用彼此内部实现 |
| Web Shared     | `apps/web/src/shared/*`   | i18n、hooks、services、utils、constants  | 放业务页面逻辑           |
| API Core       | `apps/api/src/core/*`     | 配置、环境变量、数据库、外部集成基础设施 | 放业务策略               |
| API Modules    | `apps/api/src/modules/*`  | Controller + Service 业务模块            | 直接依赖前端实现         |
| Shared Package | `packages/shared/src`     | 前后端共享类型契约                       | 放运行时业务逻辑         |

## 2. 命名约定

| 对象       | 规则                                     | 示例                       |
| ---------- | ---------------------------------------- | -------------------------- |
| 文件名     | kebab-case                               | `news-data.service.ts`     |
| React 组件 | PascalCase（导出）                       | `IntelPage`                |
| Hook       | `useXxx`                                 | `use-live-quotes.ts`       |
| 常量       | `UPPER_SNAKE_CASE` 或 `xxx.constants.ts` | `API_PATHS`                |
| i18n Key   | `domain.section.item`                    | `intel.timeline.title`     |
| DTO/类型   | `XxxRequest` / `XxxResponse`             | `PortfolioSummaryResponse` |

## 3. 前端设计约束

| 主题   | 规则                                                         |
| ------ | ------------------------------------------------------------ |
| 布局   | 信息密度优先，避免单页超长列表；长列表需分组、分页或滚动容器 |
| 视觉   | 统一 Apple 字体系；减少字体尺寸种类；间距采用固定刻度        |
| 响应式 | 默认移动优先，`md/lg` 做增强而非重写                         |
| 文案   | 禁止页面硬编码中英文；统一走 i18n                            |
| 时间   | 所有时间显示必须包含时区信息                                 |

## 4. 后端链路约束

| 主题     | 规则                                            |
| -------- | ----------------------------------------------- |
| 接口职责 | Controller 只处理 HTTP；业务逻辑在 Service      |
| 数据源   | 所有外部源通过 adapter/service 包装，必须可替换 |
| 容灾     | 外部依赖超时/失败需可降级且可观测               |
| 数据可信 | 核心数据必须带来源标识和时间戳                  |

## 5. 当前结构评审结论（2026-03-03）

| 项          | 结论                               | 处理结果                         |
| ----------- | ---------------------------------- | -------------------------------- |
| 目录分层    | 基本符合 Monorepo 分层，边界清晰   | 保持现有目录，补全文档约束       |
| 命名一致性  | 大部分符合，少量页面仍有硬编码文案 | 本次迭代推进 i18n 与常量抽离     |
| UI 可维护性 | 部分页面文件偏大，需继续拆分       | 先改核心链路，后续按域拆组件     |
| 数据真实度  | 壳层存在静态伪行情/伪新闻展示      | 本次移除静态伪内容，改为事实驱动 |
