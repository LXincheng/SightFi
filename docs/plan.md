## 4. 质量闸门（Definition of Done）

| 类别     | 规则                                           |
| -------- | ---------------------------------------------- |
| 功能完成 | 对应 PRD 验收项全部通过                        |
| 代码质量 | 无阻断级 lint/type 错误，无 `any`              |
| 测试     | 至少包含单测 + 关键流程集成测试                |
| 文档     | PRD/Plan/Rules/Log 至少更新一处                |
| 观测     | 新接口具备日志和错误追踪                       |
| 安全     | 密钥仅服务端可见，日志无明文敏感信息           |
| 弹性     | 外部依赖异常时具备可观测降级路径且不阻塞主流程 |

## 5. 风险与缓冲

| 风险             | 触发信号               | 缓冲策略                         |
| ---------------- | ---------------------- | -------------------------------- |
| 数据源授权延期   | 合同或 API Key 延迟    | 先上 Mock+免费备源，不阻塞主流程 |
| 实时链路压力高   | SSE 连接过多或延迟升高 | 增量推送 + 采样 + 连接分层       |
| UI 与 Figma 偏差 | 关键布局/样式差异明显  | 每周做一次设计回归评审           |
