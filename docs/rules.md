# SightFi 工程规约（Rules）

> 更新时间：2026-03-03  
> 目标：降低耦合、提升可维护性、控制复杂度。

## 1. 文档与流程

| 编号 | 规则 | 要求 |
| --- | --- | --- |
| R-01 | 开发前先读计划 | 开始任务前必须核对 `docs/plan.md` 当前阶段 |
| R-02 | 变更可追溯 | 关键变更必须记录到 `docs/log.md` |
| R-03 | 架构先行 | 引入新依赖/新模块前需符合 `docs/architecture.md` |
| R-04 | 配置同步 | 新增 env 或脚本必须同步 `.env.example` / README |

## 2. 架构边界

| 编号 | 规则 | 要求 |
| --- | --- | --- |
| A-01 | 前后端解耦 | 前端仅调用 BFF，禁止直连第三方 |
| A-02 | 分层清晰 | Controller 只处理 HTTP；Service 处理业务 |
| A-03 | 共享包轻量 | `packages/shared` 只放类型、常量、基础工具 |
| A-04 | 数据源可替换 | 第三方必须 adapter 化，不能硬绑定 |

## 3. 代码质量

| 编号 | 规则 | 要求 |
| --- | --- | --- |
| C-01 | TypeScript 安全 | 禁止 `any`，使用明确类型 |
| C-02 | 异常处理 | 异步逻辑必须有错误处理和用户可读反馈 |
| C-03 | 文案治理 | 页面文案统一走 i18n，禁止硬编码 |
| C-04 | 重复治理 | 同类逻辑重复超过 3 次必须抽象 |
| C-05 | 小步可回滚 | 单次改动保持最小闭环并可验证 |

## 4. UI 与体验

| 编号 | 规则 | 要求 |
| --- | --- | --- |
| U-01 | 响应式默认 | 移动端优先，再增强桌面端 |
| U-02 | 信息密度 | 避免超长单列列表，采用分组/分页/滚动容器 |
| U-03 | 字体统一 | 使用 Apple 字体系，控制字号种类 |
| U-04 | 时区可见 | 时间显示必须包含时区 |
| U-05 | 可降级动效 | 动效可关停，不影响核心功能 |

## 5. 安全与发布

| 编号 | 规则 | 要求 |
| --- | --- | --- |
| S-01 | Secret 安全 | Key 仅服务端读取，日志不输出敏感值 |
| S-02 | 发布闸门 | lint/typecheck/build/test 失败禁止合并 |
| S-03 | 降级可用 | 外部依赖异常时主链路仍可读、可用 |
