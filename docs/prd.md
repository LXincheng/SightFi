# SightFi 产品需求文档（PRD v2.1）

> 版本状态：Draft -> Baseline  
> 更新时间：2026-03-02  
> 适用范围：SightFi Monorepo（Web + BFF）

## 1. 产品概述

| 项目     | 内容                                                    |
| -------- | ------------------------------------------------------- |
| 产品名称 | SightFi                                                 |
| 产品定位 | 国际新闻 + 实时热点 + 股票/ETF 监控 + AI 研究辅助       |
| 产品形态 | Web App（PWA 可安装），支持桌面与移动端                 |
| 核心人群 | 有跨市场投资需求的个人投资者（美股/港股/ETF 优先）      |
| 核心价值 | 用“事实数据 + 组合状态 + AI 证据化结论”替代情绪化信息流 |

## 2. 问题定义与目标

### 2.1 目标问题

| 当前痛点                         | SightFi 解决方式                                |
| -------------------------------- | ----------------------------------------------- |
| 新闻信息噪声大，观点多事实少     | 新闻事实提取与来源追溯，默认展示事实摘要        |
| 多市场持仓分散，收益和风险不可视 | 统一组合账本，多视图切换（板块/市场/资产/标的） |
| AI 分析容易“说得像真的”          | 强制证据引用、置信度、风险披露、无证据降级      |
| 行情更新慢或页面卡顿             | 后端聚合缓存 + SSE 推送，客户端只请求内部 BFF   |

### 2.2 产品目标（MVP）

| 目标编号 | 描述                                      | 成功标准                          |
| -------- | ----------------------------------------- | --------------------------------- |
| G1       | 用户在 3 分钟内完成持仓录入并看到实时收益 | 首次录入成功率 >= 90%             |
| G2       | 用户可在一个页面查看事实新闻与组合影响    | 新闻条目具备来源、时间、关联资产  |
| G3       | AI 输出可追溯、可解释、不夸张             | 含证据回答占比 >= 95%             |
| G4       | 保持轻量、高可用、低耦合                  | 月可用性 >= 99.9%，架构可独立部署 |

## 3. 产品边界与合规声明

| 项目     | 规则                               |
| -------- | ---------------------------------- |
| 定位     | 研究辅助工具，不构成投资建议       |
| 不做     | 自动下单、高频策略执行、社交跟单   |
| 强制提示 | 每次 AI 建议展示风险提示与不确定性 |
| 数据争议 | 多源冲突时标注“待确认”，不静默覆盖 |

> 关键原则：先保证“信息正确与可追溯”，再追求“建议聪明”。

## 4. 设计与交互规范（结合 Figma 草稿）

> 说明：UI 以 `figma-proto` 为唯一视觉真源。PRD 仅保留产品级约束，不在此固化过细视觉实现，避免误导后续迭代。

### 4.2 信息架构（来自 Figma 原型）

| 一级模块  | 建议路由     | 核心内容                               |
| --------- | ------------ | -------------------------------------- |
| Dashboard | `/`          | 总览指标、市场热力、全球视角、异动列表 |
| Intel     | `/news`      | 客观新闻事实流、来源追溯、影响资产标签 |
| Assets    | `/portfolio` | 组合总览、持仓清单、视图切换、收益统计 |
| System    | `/settings`  | 主题、AI 严谨度、通知策略、数据偏好    |

### 4.2 UX 硬性约束

| 编号  | 约束                                                    |
| ----- | ------------------------------------------------------- |
| UX-01 | 任一关键任务（看收益、录交易、看新闻）不得超过 2 层跳转 |
| UX-02 | 移动端必须保证核心内容可读，不出现重叠和变形            |
| UX-03 | 所有实时数据必须展示“最后更新时间”                      |
| UX-04 | AI 文本与用户文本视觉分层清晰，长文本自动换行           |

## 5. 核心功能需求

### 5.1 模块 A：Objective News（客观新闻）

| 优先级 | 功能     | 描述                         | 验收标准                                       |
| ------ | -------- | ---------------------------- | ---------------------------------------------- |
| P0     | 新闻聚合 | 拉取权威源并归一化           | 新闻统一字段：`source/time/title/fact/tickers` |
| P0     | 事实提取 | 去观点化处理，只保留事件事实 | 条目可切换“事实摘要/原文链接”                  |
| P0     | 去重     | 同事件跨源聚合               | 30 分钟窗口内重复率显著下降                    |
| P1     | 影响标签 | 标注影响市场/行业/标的       | 标签含置信度与证据来源                         |

### 5.2 模块 B：Portfolio（组合账本）

| 优先级 | 功能       | 描述                                  | 验收标准                                                        |
| ------ | ---------- | ------------------------------------- | --------------------------------------------------------------- |
| P0     | 交易录入   | 买入/卖出/分红/手续费录入             | 表单可 3 步完成                                                 |
| P0     | 多视图     | 板块/市场/资产类型/标的切换           | 视图切换延迟 < 300ms                                            |
| P0     | 实时收益   | 当日、累计、仓位占比                  | 结果可与交易流水对账                                            |
| P0     | 多币种换算 | 自动将美股/港股/ETF 等资产折算到基准币 | 汇率带时间戳与来源；汇率接口异常时使用最近一次缓存并标注“汇率延迟” |
| P1     | 导入       | CSV 模板导入交易                      | 错误行提示可定位                                                |

### 5.3 模块 C：AI Research Assistant（AI 研究助理）

| 优先级 | 功能       | 描述                              | 验收标准                   |
| ------ | ---------- | --------------------------------- | -------------------------- |
| P0     | 组合体检   | 周/月报告：集中度、波动、暴露风险 | 输出固定结构，含优先级动作 |
| P0     | 证据化结论 | 每条结论必须附带数据证据          | 无证据时输出“信息不足”     |
| P0     | 规则约束   | 禁止绝对化和情绪化建议            | 命中规则自动拦截改写       |
| P1     | 记忆模块   | 风险偏好、历史错误、持仓目标      | 用户可查看/编辑/删除       |

### 5.4 模块 D：Geek Dashboard（极客监控台）

| 优先级 | 功能     | 描述                   | 验收标准                 |
| ------ | -------- | ---------------------- | ------------------------ |
| P0     | 宏观脉冲 | 指数、波动率、热点卡片 | 单屏 6-8 卡，不超载      |
| P1     | 全球视角 | 轻量热力图/地图模块    | 可关闭动画，低端设备降级 |

### 5.5 拓展模块（低耦合优先）

| 优先级 | 功能          | 描述                           | 成本评估 |
| ------ | ------------- | ------------------------------ | -------- |
| P1     | 分红/财报日历 | 基于持仓生成未来 30 天事件     | 低       |
| P1     | Quiet Alerts  | 估值分位/异动提醒（站内+邮件） | 低-中    |
| P2     | Watchlist     | 关注池与自定义分组             | 低       |

## 6. 技术架构（Monorepo，低耦合）

### 6.1 技术栈

| 层             | 选型                              | 说明                                                                 |
| -------------- | --------------------------------- | -------------------------------------------------------------------- |
| 前端           | Vite + React + Tailwind + Zustand | 快速开发，轻量状态管理                                               |
| 后端           | NestJS（BFF）                     | 对外部数据源统一封装与缓存；内置超时/重试/降级处理；禁止客户端直连第三方 |
| 数据库         | Supabase(Postgres) + Prisma       | Auth + ORM + 清晰 schema                                             |
| 缓存           | Redis                             | 行情/新闻缓存、限流、去重                                            |
| 实时           | SSE（首版）                       | 单向推送足够，复杂度低于 WebSocket                                   |
| 配置与密钥治理 | `.env` + 服务端密钥隔离           | API Key 仅后端可见、日志脱敏、Git 忽略、轮换机制与最小权限原则       |

### 6.2 Monorepo 结构（建议）

| 路径              | 说明                                  |
| ----------------- | ------------------------------------- |
| `apps/web`        | React 前端                            |
| `apps/api`        | NestJS BFF                            |
| `packages/shared` | 类型、常量、DTO、工具函数             |
| `packages/config` | TSConfig、ESLint、Tailwind 等共享配置 |
| `docs`            | PRD、计划、规则、日志                 |

### 6.3 一致性策略

| 链路          | 策略                        |
| ------------- | --------------------------- |
| 行情/新闻读取 | AP 优先，允许秒级最终一致性 |
| 交易写入/修改 | 强一致（事务 + 幂等）       |
| 用户界面      | 必须标注更新时间与数据状态  |

## 7. 数据与 AI 策略

### 7.1 数据源策略

| 层级     | 策略                                      |
| -------- | ----------------------------------------- |
| 主源     | 1-2 个权威稳定源（先稳后广）              |
| 备源     | 同类备份源，异常自动降级                  |
| 归一化   | `symbol/timezone/currency/precision` 统一 |
| 血缘追踪 | 每条记录保留 `source_id` 和抓取时间       |

### 7.2 AI 提示词与记忆约束

| 维度       | 规则                                                         |
| ---------- | ------------------------------------------------------------ |
| 投资原则   | 长期主义、反追涨杀跌、分散风险、分批止盈                     |
| 输出格式   | 结论 + 证据 + 置信度 + 风险 + 下一步                         |
| 风险控制   | 禁止“保证收益/必涨必跌/梭哈”等表述                           |
| 超时策略   | 单次 AI 请求设置超时阈值（例如 8-12s），超时立即返回降级结果 |
| 降级策略   | 主模型失败自动切备模型；全部失败回退规则引擎与模板化分析     |
| 密钥安全   | 主备 AI Key 仅服务端读取，前端不可访问；错误日志不输出完整密钥 |
| 记忆治理   | 最小化存储，支持导出与清除                                   |

## 8. 非功能需求（NFR）

| 类型     | 目标                                |
| -------- | ----------------------------------- |
| 性能     | Dashboard 首屏 TTI（P95）< 2.5s     |
| 可用性   | 月可用性 >= 99.9%                   |
| 新鲜度   | 新闻入库延迟（P95）< 60s            |
| 观测性   | 全链路日志 + 指标 + 告警            |
| 安全性   | HTTPS、JWT、RLS、敏感配置环境变量化 |
| 可维护性 | 分层架构 + 共享类型 + 最小耦合      |

## 9. 验收标准（MVP Exit）

| 编号  | 验收项             | 通过条件                     |
| ----- | ------------------ | ---------------------------- |
| AC-01 | 持仓录入与收益展示 | 用户可完成录入并看到实时收益 |
| AC-02 | 客观新闻流         | 每条新闻均可追溯来源和时间   |
| AC-03 | AI 组合总结        | 输出包含证据和风险说明       |
| AC-04 | 响应式适配         | 主流桌面/移动分辨率布局完整  |
| AC-05 | PWA                | 可安装到桌面与移动端         |

## 10. 已知风险与应对

| 风险                 | 影响               | 应对                           |
| -------------------- | ------------------ | ------------------------------ |
| 部分数据源授权成本高 | 实时能力受限       | 首版先主备 2 源，逐步升级      |
| AI 误导性陈述        | 用户决策风险       | 证据门槛 + 风险提示 + 拦截规则 |
| 实时刷新成本         | API 费用与系统压力 | 缓存分层 + 按需订阅 + 节流     |
